# Car Price Predictor

Small Flask app that predicts used car prices from a CSV and a scikit-learn pipeline.

## Prerequisites
- Python 3.10+ (you already have Python 3.14)
- Git (optional)

## Install dependencies
Run in PowerShell in the project root:

```powershell
"C:\Program Files\Python314\python.exe" -m pip install -r requirements.txt
```

If you don't have `requirements.txt`, install the main packages directly:

```powershell
"C:\Program Files\Python314\python.exe" -m pip install pandas numpy scikit-learn flask flask-cors
```

## Files of interest
- `application.py` — Flask app and routes
- `templates/index.html` — UI template (updated to populate models dynamically)
- `templates/quikr_car.csv` — uploaded dataset used to train the model
- `templates/LinearRegressionModel.pkl` — pickled pipeline model (generated by `train_model.py` or uploaded)
- `train_model.py` — script to clean the CSV and train a fresh compatible model

## Train a fresh model (recommended)
If you uploaded a dataset or want to rebuild the model to match the CSV, run:

```powershell
"C:\Program Files\Python314\python.exe" train_model.py
```

This will create/update `templates/LinearRegressionModel.pkl`.

## Run the app
Start the Flask app from the project root:

```powershell
"C:\Program Files\Python314\python.exe" application.py
```

By default it runs in debug mode on `http://127.0.0.1:5000/`.

## Test the prediction endpoint (example)
You can POST to `/predict` with form-encoded fields: `company`, `car_models`, `year`, `fuel_type`, `kilo_driven`.

Example using `curl` (PowerShell):

```powershell
curl -X POST -F "company=Hyundai" -F "car_models=Hyundai Santro Xing XO eRLX Euro III" -F "year=2007" -F "fuel_type=Petrol" -F "kilo_driven=45000" http://127.0.0.1:5000/predict
```

## Troubleshooting
- Missing model or CSV: the app shows an error banner on the index page. Ensure `templates/quikr_car.csv` and `templates/LinearRegressionModel.pkl` exist.
- Pickle incompatibility: If you encounter pickle unpickle errors due to scikit-learn version differences, run `train_model.py` to rebuild a compatible pipeline.
- If `pip` installations fail building `scikit-learn==1.6.1`, installing the latest binary wheel (`pip install scikit-learn`) usually works; the app includes a compatibility patch to unpickle older models but training a new model is safest.
